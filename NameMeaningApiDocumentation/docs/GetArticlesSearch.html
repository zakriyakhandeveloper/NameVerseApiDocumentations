Articles Search API

Live Testing
GET /api/v1/articles/search

Search articles by query string with full-text search capability across titles, subtitles, excerpts, and content.

Description

The Articles Search API provides powerful full-text search functionality across the entire articles database. Search queries are matched against article titles, subtitles, excerpts, and content fields. Results are paginated and sorted by relevance, making it easy to find the most relevant articles for any search term.

Query Parameters

q required

Search query string (required, minimum 2 characters)

Type: string | Min Length: 2 characters | Example: "name", "islamic", "meaning"
page optional

Page number for pagination (default: 1)

Type: integer | Default: 1
limit optional

Items per page (default: 10, max: 50)

Type: integer | Default: 10 | Max: 50

Request Examples

Search for "name" with limit 5:

GET /api/v1/articles/search?q=name&limit=5

Search for "islamic" with pagination:

GET /api/v1/articles/search?q=islamic&page=1&limit=10

Search for "baby naming trends":

GET /api/v1/articles/search?q=baby%20naming%20trends

cURL Examples

curl "https://namverse-api.vercel.app/api/v1/articles/search?q=name&limit=5"
curl "https://namverse-api.vercel.app/api/v1/articles/search?q=islamic&page=1&limit=10"
curl "https://namverse-api.vercel.app/api/v1/articles/search?q=meaning"

Response Fields

Field Type Description
success boolean Indicates if the request was successful
query string The search query that was executed
data array Array of matching article objects
pagination object Pagination metadata

Features

Full-text search across multiple fields
Case-insensitive search
Relevance-based sorting
Pagination support
Searches titles, subtitles, excerpts, content
Fast query execution

Error Responses

{ "success": false, "error": "Query parameter 'q' is required" }
